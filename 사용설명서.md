# 롱폼 자동화 v2 — 사용설명서

> AI가 주제만 입력하면 대본 → 이미지 → 음성 → MP4 영상까지 자동으로 만들어 주는 시스템입니다.

---

## 시작하기 전에

### 프로그램 실행 방법

1. 윈도우 탐색기에서 `C:\Users\ysc52\longform-v2` 폴더 열기
2. 주소창에 `cmd` 입력 후 엔터
3. 아래 명령어 입력 후 엔터

```
node server.js
```

4. `Server running on http://localhost:5500` 메시지가 뜨면 성공
5. 브라우저에서 `http://localhost:5500` 접속

> 서버를 끄려면 cmd 창에서 `Ctrl + C` 누르세요.

## 전체 흐름 한눈에 보기

```
STEP 0  API 키 입력
   ↓
STEP 1  영상 기본 설정 (주제, 길이, 톤 등)
   ↓

STEP 5  최종 MP4 렌더링
```

## STEP 1 — 영상 설정

### 영상 모드 선택

화면 상단 탭에서 선택합니다.

- **📺 롱폼** — 유튜브 일반 영상 (3분 ~ 30분)
- **📱 쇼츠** — 유튜브 쇼츠 세로 영상 (15초 ~ 3분)

### 입력 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| 채널 이름 | 대본 처음/끝에 자동 삽입 | 황금노년TV |
| **영상 주제** | 영상 내용 (필수) | 70대 이후 자식에게 기대지 않고 편안하게 사는 법 |
| 영상 길이 | 원하는 분량 선택 | 10분 |
| 대본 톤 | 말하는 스타일 | 따뜻하고 잔잔한 이야기형 |
| 챕터 수 | 목차 구성 (0이면 챕터 없음) | 5 |
| 대사 언어 | 한국어 / 영어 | 한국어 |

### 쇼츠 전용 옵션

- **쇼츠 스타일** 선택: 😂 웃음+반전 / 🥰 귀여움+감동 / 💡 공감+교훈
- 쇼츠 길이: 15초 / 30초 / 60초 / 90초 / 3분

### 완료

**[새 프로젝트 시작]** 버튼 클릭 → 프로젝트 ID가 상단 바에 표시됩니다.

---

## STEP 2 — 대본 생성

### 대본 생성하기

**[대본 + 메타 생성]** 버튼 클릭 → AI가 자동으로 생성합니다 (1~3분 소요)

### 생성되는 항목

| 탭 | 내용 |
|----|------|
| **대본** | 영상에서 읽을 전체 나레이션 텍스트 |
| **제목** | 유튜브 업로드용 제목 후보 10개 (클릭하면 선택) |
| **설명글** | 유튜브 영상 설명란에 붙여넣을 텍스트 |
| **해시태그** | #태그 #목록 |
| **썸네일 문구** | 썸네일 이미지에 넣을 짧은 문구 후보 |

### 대본 수정

- 대본 탭에서 직접 텍스트를 수정할 수 있습니다
- 수정 후 **[✅ 대본 완성 → 다음 단계]** 버튼 클릭

> 대본을 확정해야 다음 단계(장면 생성)가 활성화됩니다.

---

## STEP 3 — 장면 & 이미지

### 장면 생성하기

**[장면 생성]** 버튼 클릭 → AI가 대본을 장면별로 분할하고 이미지 검색어를 제안합니다

### 각 장면 카드에서 할 수 있는 것

| 기능 | 방법 |
|------|------|
| 이미지 자동 검색 | **[이미지 검색]** 버튼 클릭 → Pexels에서 자동 검색 |
| 이미지 수동 선택 | 검색 결과에서 원하는 이미지 클릭 후 **[선택 저장]** |
| 검색어 바꾸기 | 검색창 내용 수정 후 다시 검색 |
| 이미지 직접 업로드 | **[내 파일]** 버튼으로 직접 업로드 가능 |
| 영상 클립 사용 | **[영상 검색]** 탭에서 짧은 영상 클립 사용 가능 |

> Pexels 키가 없어도 이미지 없이 렌더 가능합니다 (검은 화면 + 오디오).

---

## STEP 4 — TTS 음성 생성

### 음성 생성하기

**[TTS 생성 시작]** 버튼 클릭 → 대본을 구간별로 나눠 AI 음성 생성 (수 분 소요)

- 구간당 최대 약 400자씩 나눠서 순차적으로 생성합니다
- 진행 상황이 퍼센트 바로 표시됩니다

### 생성 완료 후

- 각 구간별로 **미리듣기 플레이어**가 생성됩니다
- 상단에 **전체 오디오** 합본 플레이어와 **총 길이**가 표시됩니다
- **[전체 다운로드]** 버튼으로 WAV 파일 저장 가능

### 대본 수정 후 재생성

대본을 수정하고 다시 **[TTS 생성 시작]**을 누르면 **이전 녹음이 자동으로 사라지고** 새로 생성됩니다.

---

## STEP 5 — 최종 MP4 렌더

### 배경음악 (BGM) 설정 — 선택사항

| 방법 | 설명 |
|------|------|
| 파일 업로드 | **[📂 파일 업로드]** 버튼 → mp3/wav/m4a 파일 선택 |
| URL 입력 | 음악 파일 직접 URL 붙여넣기 후 **[불러오기]** |

무료 음악 사이트 바로가기 (화면 안에 있음):
- Pixabay Music
- Bensound
- Free Music Archive
- Incompetech
- Chosic

미리듣기 후 **[✓ 이 음악 BGM으로 사용]** 클릭 → 볼륨 슬라이더(기본 12%)로 조절

### 동영상 클립 렌더 방식

- **루프 반복** (기본): 짧은 영상 클립이 장면 내내 반복 재생
- **1회 재생 + 이미지**: 클립 1회 후 나머지는 이미지로 채움

### 쇼츠 자막 (쇼츠 모드 전용)

- 자막 오버레이 체크박스 활성화 → TTS 음성과 타이밍 맞춘 자막 자동 삽입

### 렌더 시작

**[MP4 렌더 시작]** 버튼 클릭 → ffmpeg가 이미지+음성+BGM을 합쳐 MP4 생성 (수 분 소요)

완료되면:
- 화면에서 **미리보기 재생** 가능
- **[MP4 다운로드]** 버튼으로 저장

## 프로젝트 관리

### 저장 & 불러오기

- 모든 작업은 서버의 `projects/` 폴더에 **자동 저장**됩니다
- 상단 바 **[📂 불러오기]** 버튼 → 이전 프로젝트 목록에서 선택

### 초기화

- 상단 바 **[⟳ 초기화]** 버튼 → 현재 작업 화면을 처음 상태로 리셋 (파일은 유지)

### 프로젝트 삭제

- 불러오기 모달에서 각 프로젝트 옆 **[삭제]** 버튼 클릭
- 삭제하면 TTS·이미지·영상 파일 모두 제거됨 (복구 불가)

## 자주 있는 문제

### "Gemini API 오류"가 뜰 때

- API 키가 올바른지 확인 (AIzaSy...로 시작)
- Gemini API 무료 할당량 초과 여부 확인
- 잠시 후 다시 시도 (서버 과부하 시 자동 재시도)

### TTS 생성이 너무 느릴 때

- 대본이 길수록 구간이 많아져 시간이 걸립니다
- 10분 분량이면 약 5~10분 소요됩니다

### 이미지가 안 뜰 때

- Pexels API 키가 없거나 만료된 경우
- 검색어를 영어로 바꿔보세요 (Pexels는 영어 검색이 더 정확)

### 렌더된 영상에 소리가 없을 때

- TTS 생성이 완료되었는지 확인
- `audio_full.wav` 파일이 프로젝트 폴더에 있는지 확인

### 서버가 안 켜질 때

```
cd C:\Users\ysc52\longform-v2
npm install
node server.js
```
